# Home

Welcome to specifian. Like Storybook, this tool lets you view and edit MDX files under `.specs/` as design documents in a web UI.

## How to use specifian

### front-matter becomes data
Each spec MDX file's YAML front-matter is treated as structured design data. You can access it via the `data` variable inside body React components, and fetch it all at once through the API.

```markdown
---
title: users table
table:
  name: users
  columns:
    - { name: id, type: bigint, primaryKey: true }
---

# {title}

<TableDefinition data={data.table} />
```

### Reference related specs with wiki links
Use the `[[category:slug]]` syntax to link to other specs in the same project. Dependency relationships between specs are also visualized.

```markdown
[[tables:users]] // link to .specs/tables/users.mdx
[[api:users-api|Users API]] // specify display text
```

On the graph page (`/graph`) you can interactively explore the link network.

### Edit in the Web UI
Click the "Edit" button in the left sidebar to open a split view with a Markdown editor on the left and a live preview on the right. Press Ctrl+S or click "Save" to write changes to the file.
