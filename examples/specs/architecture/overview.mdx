# System Architecture Diagram

The sample project uses a three-tier architecture: a frontend (browser), a backend API server, and a database.

## System Layout

```mermaid
graph TD
    A["Browser<br/>(React UI)"]
    B["API Server<br/>(Express)"]
    C["Database<br/>(PostgreSQL)"]
    
    A -->|HTTP/REST| B
    B -->|SQL| C
```

## ER Diagram

Shows the relationships between tables. The users table and posts table have a one-to-many relationship.

```mermaid
erDiagram
    USERS ||--o{ POSTS : "posts"
    USERS {
        bigint id PK
        string email UK
        string name
    }
    POSTS {
        bigint id PK
        bigint user_id FK
        string title
    }
```
