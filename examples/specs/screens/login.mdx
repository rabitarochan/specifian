# {title}

The first screen where account authentication takes place. The user enters their email address and password, which are then verified on the server.

## Behavior

- **Email validation** — Format check conforming to RFC 5322
- **Password validation** — Minimum 8 characters, no whitespace allowed
- **Error message on failure** — Notify the user that the email address or password is incorrect
- **Rate limiting** — Block for 5 minutes after 10 failed attempts
- **Forgot password** — Click to navigate to `/reset-password`
